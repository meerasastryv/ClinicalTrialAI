# 1. Executive Summary

Milestone 10 introduced Retrieval-Augmented Generation (RAG) capabilities into the Requirement Intelligence Engine.

The objective was to enable users to ask natural language questions about requirements and receive contextual answers generated by a Large Language Model (LLM) using information retrieved from the vector database.

This milestone integrates:

* Requirement Repository
* Vector Database (ChromaDB)
* Embedding Model
* Retrieval Engine
* Ollama Local LLM
* Prompt Engineering
* Response Generation

The solution provides an intelligent conversational interface for exploring and understanding requirements.

---

# 2. Business Objective

Traditional requirement management systems require users to manually search and read documents.

The objective of this milestone was to provide:

* Natural language interaction with requirements
* Faster requirement discovery
* Improved stakeholder productivity
* Intelligent requirement understanding
* Foundation for advanced AI-driven quality engineering capabilities

---

# 3. Functional Objectives

The Requirement Intelligence Assistant shall:

1. Accept natural language questions.
2. Retrieve relevant requirements from the vector database.
3. Provide contextual information to the LLM.
4. Generate meaningful responses using retrieved requirements.
5. Restrict responses to available requirement knowledge.
6. Operate completely on local infrastructure using Ollama.

## 5.3 Retrieval Layer

* Retrieve relevant requirement chunks

## 5.4 Local LLM

Technology:

# 6. Implementation Details

## Retrieval Process

Step 1:
Convert question into embeddings.

Step 2:
Search vector database.

Step 3:
Retrieve relevant requirement chunks.

Step 4:
Build prompt using retrieved context.

Step 5:
Submit prompt to Ollama.

Step 6:
Generate response.
